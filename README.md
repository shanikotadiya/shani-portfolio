here shown forked why?

If GitHub shows Forked from ..., it means this repository was created as a fork of another repository, not as a completely new independent repo.

✅ Why It Happens
Usually one of these:

You clicked Fork on someone else's repo/template.

You used Use this template or fork flow from another repo.

GitHub linked repository history to original source.

🔍 How to Confirm
On your repo page top area, it may say:

forked from username/repository-name
That means GitHub considers it a fork.

✅ If You Want Your Own Independent Repo
Best Fix (Recommended)
Create a brand new empty repo on GitHub (not forked), then push your local project again.

Commands:

git remote remove origin
git remote add origin https://github.com/yourusername/new-repo-name.git
git push -u origin main
✅ If You Already Have Code and Want Clean Ownership
If the current repo is forked and you don’t need fork relationship:

Option 1:
Create new repo and move code there (best)

Option 2:
Ask GitHub Support to detach fork (less convenient)

🔥 Important for Portfolio
For your personal portfolio, better to use a normal repo, not forked. It looks cleaner and more professional.
